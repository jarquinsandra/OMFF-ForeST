Files must be consistently named according to the following nomenclature:

GenusSpecie_wd_origin.txt

The program automatically detects the underscores as a new field in the database, so if you name incorrectly the files the data will not be added correctly to the database.

ID GenusSpecie WD Origin

In the case more information needs to be added to the file name you should use other symbol as "-", however since all tables in the database are automatically updated this will affect the automatization processes of the software.
For example if you add something like:

BaillonellaToxisperma_Moabi-1_WD190725_FromGabon.txt

this will cause an error in the program adding Moabi-1 as the WD number and WD190725 as the origin and FromGabon will be disregarded from the database (as seen in row 1).

ID GenusSpecie WD Origin Comments
1 BaillonellaToxisperma Moabi-1 WD190725 Wrong field order
2 BaillonellaToxisperma-Moabi WD190725 FromGabon Good field order but not included in the specie
3 BaillonellaToxisperma WD190725-1 FromGabon This would be the best option

If you add something to the GenusSpecie field, for example:

BaillonellaToxisperma-Moabi-1_WD190725_FromGabon.txt

Here the GenusSpecie field in the database will be added as BaillonellaToxisperma-Moabi and this will cause for it to be considered as a different specie from BaillonellaToxisperma and it will not be considered for the consensus spectra calculations for the specie (row 2).

The best way to include extra information is to put it at the end with a underscore separator, this will be recorded in the database in the name file and it will not mess with the automatization processes:

BaillonellaToxisperma_WD190725-1_FromGabon_Moabi.txt

The number one is added in this case to the WD number because this WD is found duplicated in the files. The best way to manage databases is to create unique WD numbers, since we have duplicated WD numbers adding the extra -1 helps to differentiate between files (row 3).
